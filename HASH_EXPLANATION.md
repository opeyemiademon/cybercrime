# Cryptographic Hashing in Digital Evidence Management

## What is a Hash?

A **cryptographic hash** is a unique digital fingerprint of a file or data. It's a fixed-length string of characters generated by running data through a mathematical algorithm. Think of it like a DNA signature for digital files - each file has a unique hash value.

## Why Do We Need Hashing in Evidence Management?

### 1. **Data Integrity Verification**
The primary purpose of hashing in digital forensics is to prove that evidence has NOT been altered, modified, or tampered with from the moment it was collected.

- If even a single bit changes in the file, the hash will be completely different
- This makes it mathematically impossible to alter evidence without detection
- Courts require proof that digital evidence is authentic and unmodified

### 2. **Chain of Custody**
In legal proceedings, you must prove:
- The evidence collected is the same evidence presented in court
- No one has tampered with it during storage, transfer, or analysis
- The hash serves as a tamper-evident seal

### 3. **Legal Admissibility**
- Digital evidence without proper hashing may be inadmissible in court
- Hash values provide cryptographic proof of authenticity
- Industry standard practice in law enforcement and cybersecurity

### 4. **Duplicate Detection**
- Identical files produce identical hashes
- Helps identify duplicate evidence across cases
- Prevents redundant storage and analysis

## SHA-256 Algorithm

### What is SHA-256?

**SHA-256** (Secure Hash Algorithm 256-bit) is part of the SHA-2 family of cryptographic hash functions designed by the NSA.

**Key Properties:**
- Produces a 256-bit (64 hexadecimal characters) hash value
- Deterministic: Same input always produces same output
- One-way function: Cannot reverse the hash to get original data
- Collision-resistant: Virtually impossible for two different files to have the same hash
- Avalanche effect: Tiny change in input creates completely different hash

**Example:**
```
File: "Hello World"
SHA-256: a591a6d40bf420404a011733cfb7b190d62c65bf0bcda32b57b277d9ad9f146e

File: "Hello World!" (added exclamation)
SHA-256: 7f83b1657ff1fc53b92dc18148a1d65dfc2d4b1fa3d677284addd200126d9069
```

Notice how adding just one character completely changes the hash!

### Hash Generation Function
Located in: `/frontend/lib/utils.ts`

```typescript
export async function generateHash(file: File): Promise<string> {
  // 1. Read the file as an ArrayBuffer (binary data)
  const arrayBuffer = await file.arrayBuffer();
  
  // 2. Use Web Crypto API to compute SHA-256 hash
  const hashBuffer = await crypto.subtle.digest('SHA-256', arrayBuffer);
  
  // 3. Convert the hash from binary to hexadecimal string
  const hashArray = Array.from(new Uint8Array(hashBuffer));
  const hashHex = hashArray.map(b => b.toString(16).padStart(2, '0')).join('');
  
  return hashHex;
}
```

### Step-by-Step Process:

1. **File Reading**: Convert the file to binary data (ArrayBuffer)
2. **Hashing**: Use browser's built-in `crypto.subtle.digest()` API with SHA-256 algorithm
3. **Conversion**: Transform binary hash to readable hexadecimal format
4. **Storage**: Save hash to database alongside evidence metadata

### At Collection Time:
```
Evidence File → Generate Hash → Store Hash in Database
```

### At Verification Time:
```
Evidence File → Generate New Hash → Compare with Stored Hash
  ↓
If Match: Evidence is authentic ✓
If Different: Evidence has been tampered with ✗
```

## Security Considerations

### Why SHA-256 Specifically?

- **Industry Standard**: Used by law enforcement worldwide
- **NIST Approved**: Recommended by National Institute of Standards and Technology
- **Collision Resistance**: No known practical attacks
- **Future-Proof**: Secure for foreseeable future (unlike older MD5 or SHA-1)

### What Hash Cannot Do:

- ❌ Cannot recover original file from hash
- ❌ Cannot prevent file deletion
- ❌ Cannot encrypt the file
- ❌ Cannot prevent unauthorized access

### What Hash CAN Do:

- ✓ Prove file hasn't been modified
- ✓ Verify data integrity
- ✓ Detect tampering
- ✓ Provide legal proof of authenticity

## Best Practices

1. **Hash Immediately**: Generate hash as soon as evidence is collected
2. **Store Securely**: Keep hash in tamper-proof database
3. **Document Algorithm**: Always specify which algorithm (SHA-256)
4. **Regular Verification**: Periodically verify evidence integrity
5. **Chain of Custody**: Log every access with hash verification
6. **Multiple Copies**: Store hash in multiple locations for redundancy

## Legal Importance

In court, the hash serves as:
- **Proof of Authenticity**: Evidence is original and unaltered
- **Expert Testimony**: Forensic experts can testify to hash matching
- **Admissibility**: Meets legal standards for digital evidence
- **Defense Against Challenges**: Opposing counsel cannot claim tampering
